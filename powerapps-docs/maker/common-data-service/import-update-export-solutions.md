# <a name="import-update-and-export-solutions"></a>ソリューションのインポート、更新、エクスポート 

 次の手順を使用して手動でソリューションをインポートできます。 信頼されているソースから取得したソリューションだけをインポートしてください。 カスタマイズには、外部ソースにデータを送信できるコードが含まれていることがあります。 既定のソリューションはそのエクスポート元である組織にのみインポートできます。別の組織にはインポートできません。  
  
1. **[[設定]](../model-driven-apps/advanced-navigation.md#settings)** > **[ソリューション]** の順に移動します。  
  
2.  ソリューションの一覧メニューで、**[インポート]** を選択します。  
  
3.  **[ソリューションのインポート]** ダイアログの **[ソリューション パッケージの選択]** 手順で、インポートするソリューションが含まれる圧縮 (.zip または .cab) ファイルを参照します。 
  
4.  **[次へ]** を選択します。  
  
5.  **[インポート]** を選択する前に、ソリューションに関する情報を表示できます。  
  
6.  ソリューションのインポートが完了するまで数分待たなければならないことがあります。 成功した場合、結果を表示したり、**[閉じる]** を選択したりできます。  
  
 発行が必要な変更をインポートした場合、先に発行しないとカスタマイズを利用できません。 
  
 インポートに失敗した場合、レポートが表示され、エラーや警告が記録されていればそれが表示されます。 **[ログ ファイルのダウンロード]** を選択すると、インポート失敗の原因に関する詳細を取得できます。 ソリューション インポート失敗の最も一般的な原因は、必要なソリューション コンポーネントの一部がソリューションに含まれていなかったことにあります。  
  
 ログ ファイルをダウンロードすると、XML ファイルが見つかります。これを Office Excel で開けば、内容を表示できます。  
  
> [!NOTE]
>  アクティブなルーティング規則セットは編集できません。 そのため、アクティブなルーティング規則セットが含まれるソリューションを同じ ID の規則が既に存在する組織にインポートすると失敗します。 詳細: [サポート案件を自動的にルーティングするルールを作成する](https://docs.microsoft.com/dynamics365/customer-engagement/customer-service/create-rules-automatically-route-cases)

## <a name="update-solutions"></a>ソリューションの更新  
 既存のマネージド ソリューションに更新プログラムをインストールすることがあります。 この手順は、新しいマネージド ソリューションのインストールに似ていますが、オプションがいくつか異なります。 他のユーザーから取得したソリューションを更新する場合、選択する必要のあるオプションに関するガイダンスをソリューションの発行元から入手する必要があります。  
  
1. **[[設定]](../model-driven-apps/advanced-navigation.md#settings)** > **[ソリューション]** の順に移動します。   
  
2.  ソリューションの一覧メニューで、**[インポート]** を選択します。  
  
3.  **[ソリューションのインポート]** ダイアログの手順 **[ソリューション パッケージの選択]** で、更新するソリューションが含まれる圧縮 (.zip または .cab) ファイルを参照します。  
4.  **[次へ]** を選択します。  
  
5.  **[次へ]** を選択する前に、ソリューションに関する情報を表示できます。 このページには黄色いバーが表示され、それに **[This solution package contains an update for a solution that is already installed]\(このソリューション パッケージには、既にインストールされているソリューションの更新プログラムが含まれています\)** と表示されます。  
  
6.  次のオプションがあります。  
  
    - **カスタマイズを維持 (推奨)**  
  
         このオプションを選択すると、コンポーネントで実行されたアンマネージド カスタマイズが維持されますが、このソリューションに含まれている更新プログラムの一部が適用されないことがあります。  
  
    - **カスタマイズを上書き**  
  
         このオプションを選択すると、このソリューションに含まれるコンポーネントで前に実行されたアンマネージド カスタマイズが上書きされます。 このソリューションに含まれるすべての更新プログラムが適用されます。  
  
     適切なオプションを選択し、**[次へ]** を選択します。  
  
7.  ソリューションのインポートが完了するまで数分待たなければならないことがあります。 成功した場合、結果を表示したり、**[閉じる]** を選択したりできます。  
  
 発行が必要な変更をインポートした場合、先に発行しないとカスタマイズを利用できません。 
  
 ソリューションの発行元は、お使いの既存のアンマネージド カスタマイズをエクスポートし、カスタマイズを上書きするオプションで発行元のマネージド ソリューションを更新し、お使いのアンマネージド カスタマイズを再インポートするように求めることがあります。 これで、ユーザーのカスタマイズを維持しながら、発行元が求めている変更が適用されます。

## <a name="export-solutions"></a>ソリューションのエクスポート  
 問題が発生してもバックアップを利用できるように、アンマネージド カスタマイズを定期的にエクスポートすることをお勧めします。 マネージド ソリューションはエクスポートできません。  
  
1. **[[設定]](../model-driven-apps/advanced-navigation.md#settings)** > **[ソリューション]** の順に移動します。   
  
2.  一覧でエクスポートするソリューションを選択し、**[エクスポート]** を選択します。  
  
3.  **[カスタマイズの公開]** 手順では、公開されたカスタマイズのみがエクスポートされることが伝えられます。また、**[次へ]** を選択する前に **[すべてのカスタマイズの発行]** オプションが表示されます。  
  
4.  ソリューションで必須コンポーネントが不足している場合、**[不足している必須コンポーネント]** 手順が表示されます。 この警告は、これをアンマネージド ソリューションとして元の組織にインポートし直す場合にのみ無視できます。 それ以外の場合、ダイアログの指示に従い、エクスポートをキャンセルし、必須コンポーネントを追加してください。  
  
5.  **[システム設定のエクスポート (詳細)]** 手順では、ソリューションに含める特定のシステム設定を選択できます。 ソリューションがシステム設定のグループに依存する場合、それを選択し、**[次へ]** を選択します。  
  
     各オプションで追加される設定の詳細については、下の「**ソリューション エクスポートの設定オプション**」を参照してください。  
  
6.  **[パッケージの種類]** 手順では、**[アンマネージド]** または **[マネージド]** ソリューションとしてソリューションをエクスポートするように選択する必要があります。  
  
7.  次の手順では、特定の Dynamics 365 バージョンのターゲット ソリューションを選択できます。 このオプションは通常、前のバージョンとの間で互換性があるソリューションのエクスポートを希望する ISV によって使用されます。 自分が使用している組織バージョンと同じバージョンにアップグレードされていない組織にこのソリューションをインポートしない限り、既定値をそのまま選択してください。   
  
8.  **[エクスポート]** を選択し、ソリューション ファイルをダウンロードします。  
  
 厳密にはファイルのダウンロード動作はブラウザーによって異なります。
